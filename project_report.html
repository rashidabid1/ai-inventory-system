1. Executive Summary & Project Overview

In the modern landscape of logistics and resource coordination, legacy spreadsheet applications fail to deliver real-time operational efficiency and data transparency. **AInventory** resolves these deficits by presenting a comprehensive, secure ERP and analytics application that enables warehouse operators to manage stock, analyze cost structures, record dynamic sales depletion, and query data logs using natural language.

By implementing a robust backend with Express and Mongoose alongside a gorgeous glassmorphism frontend engineered in Vite and React, the system presents fluid interactions designed to engage administrators. With multi-tenant architecture secured by **Clerk Google Social Login** and natural language query pipelines backed by the **Google Gemini 2.5 Flash SDK**, AInventory shifts standard static data metrics into highly responsive, executive-level insights.

3. Full-Stack MERN Architecture Specs

3.1 Hybrid Social Authentication Model

To achieve high levels of workspace isolation while enabling lightweight local developer setup, a **Hybrid Auth Adapter Pattern** was implemented.

The wrapper `AuthProvider.jsx` reads `VITE_CLERK_PUBLISHABLE_KEY`. If this key is absent, the system boots into **Developer Guest Mode** by injecting a mock user configuration (`Rashid Abid`). If the key is detected, it securely loads Clerk's React SDK. The `useAppAuth.js` adapter decouples this toggle from components like `Sidebar.jsx`, presenting a unified API (`isSignedIn`, `user`, `signOut()`) regardless of the active authentication provider.

3.2 Real-time Financial Profit Extraction

Instead of standard static inventory logging, the **Sales Ledger Engine** calculates profitability in real-time. When a transaction is submitted, the Express route fetches the targeted Product document, verifies whether the requested quantity is available, extracts the cost price, computes the total sale cost, subtracts it from the selling price, and registers the remaining sum directly into the `profitPKR` field of the `Sale` schema. This allows the frontend dashboard to display absolute gross margins immediately.

3.3 Secure Backend API Decoupling

All network communication is dynamically resolved by the frontend reading `import.meta.env.VITE_API_URL` with a standard fallback to `http://localhost:5000`. Environment keys are completely isolated on the server-side, protecting your Mongoose connections and the Google Gemini API keys from browser inspection.

4. API Endpoint Specifications (Extended REST)

GET /api/products

Fetches all products inside the Mongoose catalogue. Ideal for loading ledgers or populating select inputs.

POST /api/products

Registers a new product document. Validates unique SKU parameters prior to MongoDB commit.

POST /api/sales

Submits a new customer sale. Verifies stock, depletes Product document quantity, and registers exact gross profit values.

POST /api/ai/query

Core analytical endpoint. Summarizes current products, recent sales logs, total capital, and low stock items before transmitting the dataset to Gemini 2.5 Flash. Stores user query and assistant outputs directly in the `AiLog` schema.

5. Database Schema Specifications (Mongoose)

Below is the explicit code implementation of Mongoose Schemas configured inside your Express backend:

5.1 Product Schema Specification

const mongoose = require('mongoose');

const productSchema = new mongoose.Schema({
  name: { type: String, required: true },
  sku: { type: String, required: true, unique: true },
  quantity: { type: Number, required: true, default: 0 },
  costPricePKR: { type: Number, required: true },
  sellingPricePKR: { type: Number, required: true },
  category: { type: String, required: true },
}, { timestamps: true });

module.exports = mongoose.model('Product', productSchema);

5.2 Sale Schema Specification

const mongoose = require('mongoose');

const saleSchema = new mongoose.Schema({
  productId: { type: mongoose.Schema.Types.ObjectId, ref: 'Product', required: true },
  quantity: { type: Number, required: true },
  sellingPricePKR: { type: Number, required: true },
  totalAmountPKR: { type: Number, required: true },
  profitPKR: { type: Number, required: true },
  date: { type: Date, default: Date.now }
});

module.exports = mongoose.model('Sale', saleSchema);

5.3 Assistant Conversation Log Schema Spec

const mongoose = require('mongoose');

const aiLogSchema = new mongoose.Schema({
  query: { type: String, required: true },
  response: { type: String, required: true },
  timestamp: { type: Date, default: Date.now }
});

module.exports = mongoose.model('AiLog', aiLogSchema);

6. User Manual & Operations Guide

6.1 Accessing the Platform

When entering the live environment, you are prompted with the secure Sign-In gate. Click **"Continue with Google"** to establish a secure link. If running in a local setting without a Clerk token, the interface redirects automatically to the Dashboard under a secure Guest session.

6.2 Conversing with the AI Assistant

Navigate to the **AI Assistant** panel. In the dialogue input, enter questions like: "Which products are currently low in stock and what are their values?" The chatbot communicates with the Gemini engine and returns custom, beautiful HTML tables detailing names, quantities, and values in real-time.

6.3 Dynamic Data Exports

Go to the **Settings** view. Under the **Document & Export Hub**, select your preferred document type:

  • Download PDF Ledger: Dynamic PDF generation displaying key KPIs and catalog margins.
  • Export Excel Spreadsheet: UTF-8 CSV spreadsheet generation, immediately parsed by Excel.
